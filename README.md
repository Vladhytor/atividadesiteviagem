## Descrição
Esta atividade consiste na criação de um website de turismo multi-páginas denominado **ExploreMundo**. O projeto apresenta uma página principal com destaques de destinos e uma página secundária detalhada com novas opções de lazer, como parques e reservas. O layout foi desenvolvido com foco em organização visual moderna e navegação fluida entre arquivos.

## Objetivo
Praticar conceitos de desenvolvimento web front-end, incluindo:
- Estrutura de navegação entre múltiplas páginas HTML (`index.html` e `destinos.html`).
- Estilização com **CSS3 externo** para padronização visual.
- Gestão de mídias e caminhos relativos na raiz do projeto.
- Criação de formulários de contato e seções informativas.
- Publicação de ambientes estáticos com **GitHub Pages**.

## Requisitos da Atividade
Durante o desenvolvimento do projeto, foram seguidas as seguintes etapas:

1. Criar uma estrutura de arquivos contendo páginas HTML e folha de estilo CSS.
2. Desenvolver uma página principal (`index.html`) com seção *Hero*, cards de destinos populares e formulário de contato.
3. Criar uma página adicional (`destinos.html`) para listar categorias específicas como Reservas Ecológicas e Parques Temáticos.
4. Implementar navegação funcional entre as páginas através de menus e botões de ação.
5. Utilizar **CSS3** para o layout, aplicando Flexbox para o alinhamento dos elementos.
6. Organizar as imagens na raiz do diretório para facilitar o deploy e a leitura do código.
7. Realizar o versionamento do código utilizando Git.
8. Criar um repositório no GitHub e publicar o site via **GitHub Pages**.

## Tecnologias Utilizadas
- **HTML5**
- **CSS3**
- **Git**
- **GitHub**
- **GitHub Pages**

## Publicação
O website foi publicado utilizando **GitHub Pages**, permitindo o acesso e visualização da interface diretamente pelo navegador.

## Observações
- O projeto foi desenvolvido separando o conteúdo (HTML) da estilização (CSS).
- A navegação entre as páginas foi testada para garantir que o usuário possa ir e voltar dos destinos sem erros.
- As imagens foram padronizadas na raiz do repositório para assegurar o carregamento correto após a publicação.
